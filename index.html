Output Framework

공통 개발 절차와 산출물별 작성 기준의 분리

자료 간 직접적인 모순은 크지 않으나, 팀 발표용 자료와 개인 보고서용 자료, 기존 워크숍형 템플릿과 2026 베트남 표준안 사이에는 강조점과 범위의 차이가 있다. 본 가이드는 2026 Vietnam Action Plan Development Standard를 상위 구조로 삼고, 기존 Action Plan Guideline은 팀 발표의 보조 질문으로, Individual Policy Report Guideline은 개인 보고서의 작성 기준으로 반영한다.

A

팀 Action Plan Presentation

공통 정책문제에 대한 간결한 의사결정 브리핑이다. 문제정의, 대안 비교, 실행 로드맵, KPI, 의사결정 요청을 중심으로 구성한다.

B

Individual Policy Report

개별 부처·기관의 정책문제에 대한 분석적 문서 과제이다. 상황 분석, 분석틀, 대안 평가, 권고안, 모니터링 및 평가 체계를 포함한다.

Development Standard

공통 개발 절차: Problem → Evidence → Options → Implementation → Results

1. 문제정의 누가, 어디서, 어떤 부정적 상태를 겪는지 좁힙니다.
2. 근거진단 데이터, 현장 관찰, 이해관계자 의견으로 원인을 확인합니다.
3. 한국사례 번역 1-3개 사례를 adopt / modify / avoid 기준으로 해석합니다.
4. 대안비교 최소 2-3개 대안을 효과성, 비용, 실현가능성으로 비교합니다.
5. 실행계획 책임기관, 일정, 예산, 파트너십, 산출물을 지정합니다.
6. 성과관리 1개 goal, 최대 3개 objectives, 3-5개 KPI를 연결합니다.
통합 기준 baseline 반영 방향
자원·실행 조건 Institutional/Legal, Budget/Financing, Human Resources/Capacity, Data/Technology/Infrastructure, Communication/Partnerships의 5대 enabler로 정리한다.
기술 요소 디지털·AI 정책은 technical architecture를 포함하고, 비기술 정책은 institutional design 또는 service delivery model로 대체한다.
시간 범위 기본 로드맵은 0-12개월 실행계획으로 제시하고, 필요한 경우 2-5년 scale-up plan을 별도로 제시한다.

Individual Report

개인 보고서: 소속 기관의 정책문제에 대한 분석적 권고

1. Executive Summary

문제, 핵심 분석, 권고안을 1쪽 내외로 요약합니다.

2. Introduction

정책 배경, 주제 선정 이유, 보고서 목적을 제시합니다.

3. Problem Definition

핵심 문제, 규모, 이해관계자, 미대응 시 위험을 정리합니다.

4. Current Situation Analysis

정책, 제도, 조직, 예산, 기술 상황과 기존 제도의 한계를 분석합니다.

5. Policy Options

최소 2-3개 대안을 제시하고 비교 기준을 명확히 합니다.

6. Recommendation & M&E

선택 대안, 실행전략, KPI, 모니터링 및 환류 체계를 제시합니다.
