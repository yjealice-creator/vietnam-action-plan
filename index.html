Structure Decision

팀 발표와 개인 리포트는 쪼개되, 앞단의 학습 흐름은 같이 갑니다

검토 의견: GPT가 제안한 구조는 방향이 좋습니다. 다만 웹페이지에서는 처음부터 팀 발표와 개인 리포트를 완전히 분리하면 중복이 많아지고, 완전히 합치면 산출물의 성격 차이가 흐려집니다. 따라서 공통 절차는 하나로 두고, 최종 산출물 섹션에서 팀 발표와 개인 리포트를 분리하는 방식이 가장 적합합니다.

A

팀 Action Plan Presentation

15분 내외의 의사결정자 브리핑입니다. 팀은 하나의 공통 정책문제를 선택하고, 승인 또는 지원을 요청할 수 있는 실행계획으로 제시합니다.

B

Individual Policy Report

문서형 정책보고서입니다. 개인은 자기 부처나 기관의 실제 정책문제를 분석하고, 근거 기반 대안 비교와 권고안을 제시합니다.

Main Root

공통 작업 절차: Problem → Evidence → Options → Implementation → Results

1. 문제정의 누가, 어디서, 어떤 부정적 상태를 겪는지 좁힙니다.
2. 근거진단 데이터, 현장 관찰, 이해관계자 의견으로 원인을 확인합니다.
3. 한국사례 번역 한국 제도를 복사하지 않고 설계 원리를 베트남 맥락에 맞춥니다.
4. 대안비교 최소 2-3개 대안을 효과성, 비용, 실현가능성으로 비교합니다.
5. 실행계획 책임기관, 일정, 예산, 파트너십, 산출물을 지정합니다.
6. 성과관리 Output, Outcome, Impact와 KPI, 위험 완화방안을 제시합니다.

Individual Report

개인 리포트는 자기 기관의 정책문제를 분석하는 문서형 보고서입니다

1. Executive Summary

문제, 핵심 분석, 권고안을 1쪽 내외로 요약합니다.

2. Introduction

정책 배경, 주제 선정 이유, 보고서 목적을 제시합니다.

3. Problem Definition

핵심 문제, 규모, 이해관계자, 미대응 시 위험을 정리합니다.

4. Current Situation Analysis

정책, 제도, 조직, 예산, 기술 상황과 기존 제도의 한계를 분석합니다.

5. Policy Options

최소 2-3개 대안을 제시하고 비교 기준을 명확히 합니다.

6. Recommendation & M&E

선택 대안, 실행전략, KPI, 모니터링 및 환류 체계를 제시합니다.
